# Как участвовать в проекте

Спасибо за интерес к VikaVoice! Проект на концепт-стадии: документация — основной артефакт,
код — скелет. Вклад в доки ценится не меньше кода.

## Языки

- **Документация** — русский (целевая аудитория и compliance-контекст — РФ).
- **Код, идентификаторы, commit-scope** — английский; комментарии и docstrings — русский допустим.

## Коммиты

Используем [Conventional Commits](https://www.conventionalcommits.org/ru/):

```
<type>(<scope>): <кратко, в повелительном наклонении, на русском>
```

Типы: `feat`, `fix`, `docs`, `chore`, `refactor`, `test`, `ci`.
Примеры из истории репозитория: `docs: MULTI-DEVICE — несколько Вик как единая система`,
`fix(software): заголовок ingest обязателен первым сообщением`.

## Pull Requests

1. Ветка от `master`, имя: `<type>/<кратко>` (например `docs/adr-0010-diarization`).
2. Заполните шаблон PR (чеклист: docs обновлены? лицензии затронуты? нужен ADR?).
3. CI должен быть зелёным: markdownlint, проверка ссылок, `ruff`, `pytest`.

## Архитектурные решения (ADR)

Любое решение уровня «платформа / протокол / движок / лицензия» оформляется как ADR
в [`docs/adr/`](docs/adr/README.md) (формат MADR). Процесс:

1. Создайте issue по шаблону «ADR proposal» или сразу PR с файлом `NNNN-<slug>.md`, статус `proposed`.
2. Обсуждение — в PR. Принятие — merge со статусом `accepted`.
3. Отменяющее решение не редактирует старый ADR, а создаёт новый со ссылкой `supersedes`.

## Правило одного источника истины

Каждая цифра (цена, модель платы, порт, latency-цель) живёт ровно в одном документе,
остальные ссылаются. Прежде чем добавить число — проверьте, нет ли его уже
(`docs/hardware/`, `docs/reference/`).

## Данные и приватность

- **Никогда** не коммитьте аудиозаписи, стенограммы, голосовые отпечатки и токены —
  это персональные (в т.ч. биометрические) данные. `.gitignore` защищает от очевидных
  случаев (`*.wav`, `data/`, `.env`), но ответственность на авторе PR.
- IP-адреса, хосты и учётные данные инфраструктуры в документацию не попадают.

## Локальная разработка (software/)

```bash
cd software
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
pytest              # тесты должны проходить
ruff check .        # линтер
```
