# 0004. Движок распознавания речи (ASR)

- Статус: **accepted**
- Дата: 2026-07-07

## Контекст и проблема

Нужен ASR с хорошим русским, работающий локально на устройстве (Edge, приватный режим)
и в облаке (SaaS). Редакции требуют выбора движка конфигурацией, а не пересборкой.

## Рассмотренные варианты

| Вариант | RU-качество | Локально на RPi | Лицензия | Примечание |
|---|---|---|---|---|
| **whisper.cpp** (tiny/base/medium/large) | хорошее (base) → отличное (large) | да, base на RPi4/5 | MIT (код и веса) | сервер из Meetily, порт 8178 |
| Parakeet (NVIDIA NeMo, Rust-движок из Meetily) | хорошее | лучше на Jetson/GPU | код ок, **веса — условия NVIDIA уточнить** | кандидат для толстого тира с ускорителем |
| faster-whisper (CTranslate2) | = whisper | да | MIT | альтернативная обвязка, рассмотреть при интеграции |
| Vosk | среднее | да, очень лёгкий | Apache-2.0 | fallback для слабого железа |
| Облачные API (Yandex SpeechKit и др.) | отличное | нет | SaaS | только Cloud-редакция; данные уходят провайдеру |

## Решение

- **Основной локальный движок — whisper.cpp** (сервер из вендоренного Meetily), модель по
  платформе: `base` — по умолчанию на RPi4/RPi5 (баланс скорость/качество), `medium+` —
  при наличии ускорителя или на сервере. Язык — `ru`.
- **Единый интерфейс `ASRBackend`** (`software/core/asr/base.py`): реализация выбирается
  конфигом (`whisper` | `remote`; `parakeet` — запланирован). Никакой части системы не
  позволено зависеть от конкретного движка.
- **Cloud-редакция** может использовать внешний ASR через `RemoteAPI` — выбор провайдера
  отдельным решением с учётом 152-ФЗ (данные уходят провайдеру).
- Parakeet — кандидат после выяснения условий на веса (см. THIRD_PARTY_NOTICES).

## Последствия

- Метрика качества (WER на собственном русском тестовом наборе) обязательна до споров о
  движках — задача в EPIC-1.
- Заявления маркетинга о качестве привязываются к фактически используемой модели
  (не «large-v3 из коробки», если на устройстве работает `base`).
