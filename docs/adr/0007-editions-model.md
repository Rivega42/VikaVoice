# 0007. Модель редакций: одно ядро → Edge / Cloud / On-prem / гибрид

- Статус: **accepted**
- Дата: 2026-07-07

## Контекст и проблема

Целевые сегменты предъявляют взаимоисключающие требования к месту обработки данных:
юристы/госсектор — «ничего не покидает периметр», массовый сегмент — «дёшево и без
своего сервера». Держать несколько кодовых баз — нельзя.

## Рассмотренные варианты

1. **Отдельные продукты** (локальный и облачный) — двойная стоимость разработки, дрейф
   функциональности. Отклонено.
2. **Только облако** — закрывает главное преимущество (локальность) и половину сегментов.
   Отклонено.
3. **Одно ядро, редакция выбирается конфигурацией** (env/compose-профили): те же сервисы,
   различие — где они запущены и куда пишут данные.

## Решение

Вариант 3. Матрица (детали — [concept/editions.md](../concept/editions.md)):

| Редакция | Железо | Обработка | Данные |
|---|---|---|---|
| **Edge** | толстое устройство (RPi5) | на устройстве | локально |
| **Cloud (SaaS)** | тонкое устройство (Zero 2W) | наше облако | наше облако |
| **On-premise** | тонкое или толстое | контур заказчика | контур заказчика |
| **Гибрид** (режим) | любое | realtime локально, «чистовая» — по выбору | по выбору на встречу |

Технические правила:

- Различия редакций выражаются **только** переменными окружения и compose-профилями
  (`EDITION=edge|cloud|onprem`, профили `with-asr`, `with-llm`) — см.
  [deployment](../architecture/deployment.md).
- Функциональные ветвления в коде по редакции запрещены, допускаются по **возможностям**
  (есть ли локальная LLM, доступна ли сеть) — тогда гибрид и приватный режим получаются
  бесплатно.

## Последствия

- CI собирает один набор образов; редакция — конфигурация поставки.
- Матрица «фича × редакция» поддерживается в concept/editions.md как источник истины
  для продаж.
