# 0005. Движок синтеза речи (TTS)

- Статус: **accepted**
- Дата: 2026-07-07

## Контекст и проблема

Голос Вики должен звучать с устройства с малой задержкой, работать офлайн (приватный
режим) и иметь пригодный русский. Клонирование/фирменный голос — желательная перспектива.

## Рассмотренные варианты

| Движок | Латентность | Качество RU | Работает офлайн на RPi | Лицензия |
|---|---|---|---|---|
| **Piper** (голоса irina, ruslan) | ~десятки мс, CPU | хорошее | да | MIT (код); лицензию конкретного голоса проверить |
| CosyVoice 2 | ~150 мс, нужен GPU | отличное + клонирование | нет (сервер) | Apache-2.0 |
| Silero TTS | низкая, CPU | хорошее | да | у моделей нестандартные условия — проверить |
| Облачные API (OpenAI TTS и др.) | 300+ мс + сеть | отличное | нет | SaaS |

## Решение

- **MVP и приватный режим: Piper** (CPU, офлайн, MIT), русский голос `irina` или `ruslan` —
  финальный выбор по субъективному тесту в EPIC-5.
- Стриминг по предложениям (начинать озвучивать первый фрагмент ответа, не дожидаясь
  всего текста) — обязательная оптимизация латентности.
- **Перспектива:** фирменный голос Вики через CosyVoice 2 на серверной стороне
  (Cloud-редакция) — отдельная задача после MVP; на устройстве остаётся Piper.

## Последствия

- TTS-абстракция в клиенте устройства (аналогично `ASRBackend`), чтобы смена движка не
  трогала диалоговый цикл.
- Лицензии весов конкретных голосов Piper фиксируются в THIRD_PARTY_NOTICES при интеграции.
