# AP-режим первичной настройки

> Статус: 🔲 EPIC-7 — обязательно перед продажей устройств · Обновлено: 2026-07-07

## Идея

Пользователь покупает Вику, включает — она **не должна** требовать SSH или клавиатуру.
Критерий: распаковать и запустить за 5 минут.

## Механика

1. При первом запуске устройство поднимает свой Wi-Fi: `VikaVoiceSetup_XXXX`
2. На экране / LED-кольце показывает **QR-код** для подключения
3. Пользователь подключается телефоном → попадает на `http://192.168.4.1`
4. Вводит: имя своего Wi-Fi + пароль + токен OCPlatform + (опц.) выбор голоса Вики
5. Устройство перезагружается, подключается к домашней сети, готово к работе

## Технический подход

- `hostapd` + `dnsmasq` для AP-режима (встроенный Wi-Fi чип RPi поддерживает)
- Простой FastAPI веб-интерфейс на `192.168.4.1:80`
- Конфиг хранится в `/etc/vikavoice/config.yaml`
  ([reference/configuration.md](../reference/configuration.md))

## Дополнительно

- Тот же UI для перенастройки: `http://vika.local/settings` через mDNS
- Индикация «нужна настройка» — жёлтый пульс LED
- Второе устройство: тот же AP-режим или автоподключение через hub
  (см. [multi-device](../architecture/multi-device.md#открытые-вопросы))

## Безопасность

Setup-AP открыт по определению — окно атаки. Требования: AP активен только до первой
настройки (и по физической кнопке reset), токен OCPlatform вводится один раз и хранится
с правами 600, страница настройки недоступна из основной сети после конфигурации.
См. [модель угроз](../architecture/security-threat-model.md).
