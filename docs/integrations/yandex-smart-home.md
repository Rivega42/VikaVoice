# Умный Дом Яндекс (УДЯ)

> Статус: 🔲 backlog (EPIC-8) · Обновлено: 2026-07-07

## Идея

Управлять умным домом голосом через Вику, не через Алису. Читать состояние датчиков
(температура, CO2, окно, обогреватель) и использовать это в контексте разговора.

**Примеры:**

- «Вика, включи свет на кухне» → Вика дёргает API УДЯ
- «Вика, как дома?» → берёт CO2, температуру, статус окна и отвечает голосом
- Проактивно: «CO2 634 ppm, стоит проветрить»
- Сценарии: «Вика, я пришёл» → свет + сценарий проветривания

## Технический подход

Яндекс отдаёт **OAuth-токен на 365 дней** для чтения/управления УДЯ.

**Механика получения токена** (по мотивам статей на Habr, см. источники):

1. Регистрируем «приложение» на https://oauth.yandex.ru — scope `iot`
2. Пользователь идёт по ссылке, вводит код с экрана Вики, подтверждает
3. Токен сохраняется в конфиг устройства ([configuration.md](../reference/configuration.md))
4. Автопродление за N дней до истечения

**API:**

- `GET https://api.iot.yandex.net/v1.0/user/info` — список устройств и сценариев
- `POST .../devices/actions` — управление
- `GET .../devices/states` — текущие показания

**Куда встроить:**

- В OCPlatform как инструменты `smart_home.query`, `smart_home.action`
- В приватном режиме — недоступно (нужен интернет); честно сообщать пользователю
