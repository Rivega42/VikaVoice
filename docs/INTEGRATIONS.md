## 🏡 Умный Дом Яндекс (УДЯ)

### Идея
Управлять умным домом голосом через Вику, не через Алису.
Читать состояние датчиков (температура, CO2, окно, обогреватель) и использовать это в контексте разговора.

**Примеры:**
- «Вика, включи свет на кухне» → Вика дёргает API УДЯ
- «Вика, как дома?» → берёт CO2, температуру, статус окна и отвечает голосом
- Проактивно: «Ром, CO2 634 ppm, стоит проветрить»
- Сценарии: «Вика, я пришёл» → включает свет + сценарий проветривания

### Технический подход
Яндекс отдаёт **OAuth-токен на 365 дней** для чтения/управления УДЯ.

**Механика получения токена** (позаимствовано из habr 1052388):
1. Регистрируем «приложение» на https://oauth.yandex.ru — scope `iot`
2. Пользователь идёт по ссылке, вводит код с экрана Вики, подтверждает
3. Токен сохраняется в конфиг Вики
4. Автопродление за N дней до истечения

**API:**
- `GET https://api.iot.yandex.net/v1.0/user/info` — список устройств и сценариев
- `POST .../devices/actions` — управление
- `GET .../devices/states` — текущие показания

**Куда встроить:**
- В OCPlatform как инструмент (`smart_home.query`, `smart_home.action`)
- В приватном режиме — недоступно (нужен интернет)

## 📱 Мини-экран-компаньон (ESP32)

### Идея
Помимо основного 7" экрана в продуктовой версии — сделать **дешёвого компаньона** на базе GeekMagic SmallTV Pro (ESP32) или аналога.

**Зачем:**
- Стоит на столе рядом, показывает состояние Вики
- Индикация: «слушает» / «думает» / «говорит» / **красный «приватный режим»**
- Если сама Вика в другой комнате — компаньон рядом, чтобы видеть статус
- Стоит 300–700 ₽ вместо 4–6к за большой экран

**Что показывает:**
- Иконку состояния Вики
- Текущее время
- (опц.) короткий текст последней реплики
- (опц.) индикатор аудио-активности

**Технический подход:**
- ESP32 подключается по WiFi к Вике
- Опрашивает `GET /status` каждые 500мс или подписывается на WebSocket
- Своя прошивка на Arduino-стеке
- Идея форм-фактора: маленький кубик или дисплей-часы на столе

### Кандидаты железа (~300–700 ₽)
- **GeekMagic SmallTV Pro** — ESP32-WROOM-32, IPS-экран, WiFi, AliExpress
- **NEIMZ мини (NZ1237)** — клон SmallTV Ultra, ~329 ₽
- **GeekMagic GIFTV Crystal Cube** — та же начинка в стеклянном кубике, эффектнее, но тусклее

### Источники
- https://habr.com/ru/articles/1025884/ — статьи автора про эти железки
- https://habr.com/ru/articles/1052388/ — прошивка YaMon (можно использовать её структуру как reference)

### Статус
🔲 Backlog — после продуктовой версии основного устройства.

## 🛜 AP-режим первичной настройки

### Идея (важно для продуктовой версии)
Пользователь покупает Вику, включает — она **не должна** требовать SSH или клавиатуру.

**Механика** (позаимствовано из YaMon):
1. При первом запуске Вика поднимает свой WiFi: `VikaVoiceSetup_XXXX`
2. На экране / LED-кольце показывает **QR-код** для подключения
3. Пользователь подключается телефоном → попадает на `http://192.168.4.1`
4. Вводит: имя своего WiFi + пароль + токен OCPlatform + (опц.) выбор голоса Вики
5. Вика перезагружается, подключается к домашней сети, готова к работе

**Технический подход:**
- `hostapd` + `dnsmasq` для AP-режима на RPi4 (встроенный Wi-Fi чип поддерживает)
- Простой Flask/FastAPI веб-интерфейс на `192.168.4.1:80`
- Конфиг хранится в `/etc/vikavoice/config.yaml`

**Дополнительно:**
- Тот же UI для перенастройки: `http://vika.local/settings` через mDNS
- Индикация «нужна настройка» — жёлтый пульс LED

### Статус
🔲 M4 (продуктовая версия) — обязательно перед продажей устройств.

## 📅 Календарь и задачи

### Идея
Вика знает расписание, напоминает, планирует, добавляет задачи по голосу.

**Кандидаты интеграций:**
- Google Calendar
- Яндекс.Календарь
- Notion
- Todoist
- Trello (для команд — на встречах)

Уже частично работает через OCPlatform-инструменты. В VikaVoice просто пробрасываем.

### Статус
✅ Работает через OCPlatform, отдельной работы не требуется.

## Не берём

- **Прошивка YaMon целиком** — она под ESP, у нас RPi4/Linux. Полезны идеи (OAuth-токен, AP-режим), сам код нет.
- **Alice SDK** — не хотим зависеть от Яндекса как платформы, только использовать УДЯ как источник данных.
- **Home Assistant** — большая экосистема, потенциально можно подключить как дополнительный «мост» к куче устройств, но не приоритет.
