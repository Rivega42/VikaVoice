# Мобильный клиент — Вика в наушниках через телефон

> Сценарий: телефон + любые наушники (Bluetooth или проводные) = полноценная Вика в кармане.
> Идёшь по улице, едешь в машине, в спортзале — Вика на связи, память та же что дома.

## Ключевая идея

**Телефон = ещё один VikaVoice-satellite**, наравне с домашними RPi4-устройствами.

Всё через ту же **OCPlatform-сессию** — Вика помнит что ты сказал утром дома,
отвечает то же самое в наушниках в машине. Никакой разрыв контекста между «домом»
и «улицей».

---

## Варианты подключения — от простого к продвинутому

### 1. Telegram голосовыми 🟢 (уже работает)

**Как:** тапаешь «микрофон» → говоришь → отправляешь → Вика транскрибирует и отвечает.
С директивами `[[tts:...]]` — голосом. Слышишь через наушники Telegram.

| Плюсы | Минусы |
|-------|--------|
| Работает прямо сейчас | Не hands-free (нужен tap) |
| 0 разработки | Не realtime разговор |
| Все наушники поддерживаются | Задержка |

**Статус:** ✅ доступно. Используй как fallback.

---

### 2. PWA (Progressive Web App) 🌐 (первый шаг)

**Как:** отдельный сайт `vika.grandhub.ru`. Открыл в браузере на телефоне → работает.
Можно «установить» на главный экран как приложение.

**Технологии:**
- Web Speech API — STT + TTS прямо в браузере
- Fetch к OCPlatform gateway для мозга
- Service Worker для оффлайн-подсказок

| Плюсы | Минусы |
|-------|--------|
| Быстро сделать (1-2 недели) | Wake-word в фоне не работает |
| Никаких сторов | Только tap-to-talk |
| Работает и iOS, и Android | Качество Web Speech хуже нативного |
| Легко обновлять | На iOS ограничения Safari |

**Статус:** 🔲 M2 (после MVP домашнего устройства).

---

### 3. Нативное приложение 📱 (полноценный клиент)

**Как:** Flutter или React Native → App Store + Google Play.

**Фичи:**
- Wake-word «Вика» работает **в фоне** (Porcupine SDK для мобилок)
- Наушники подключены к телефону как обычно (BT/провод)
- STT/TTS локально на телефоне (Speech Framework iOS / Google Speech Android)
- Мозг через OCPlatform gateway
- Fallback на телефонный микрофон если наушники не пустили wake-word

**Приватный режим на телефоне:**
- Локальная маленькая LLM (Gemma 3n / Phi-3.5-mini через MediaPipe/ONNX Runtime)
- Или просто "приглушение" — только STT, ответ показать текстом, не идти в облако

| Плюсы | Минусы |
|-------|--------|
| Полноценный hands-free | Нужно писать (2-3 месяца) |
| Wake-word в фоне | Публикация в стор |
| Работает везде где 4G | Поддержка iOS + Android |
| Батарея дольше PWA | Комиссии Apple/Google |

**Статус:** 🔲 M4-M5 (после базовой функциональности).

---

### 4. Bluetooth headset прямо к устройству 🎧

**Как:** твои наушники подключаются **не к телефону**, а к чему-то что связано с Викой:
- К домашнему VikaVoice-устройству (когда дома)
- К мини-хабу в кармане: LTE-модем + RPi Zero 2W + BT (когда на улице)

**Плюсы:**
- Реальный «агент в наушниках», без телефона в руках
- Работает даже когда телефон в кармане с выключенным экраном
- Домашние наушники автоматически подключаются к домашней Вике

**Минусы:**
- BT range ограничен (~10м)
- Нужно отдельное носимое устройство (мини-хаб)
- Сложнее для пользователя (два девайса)

**Статус:** 🔲 Backlog. Интересный форм-фактор для будущего.

---

### 5. WebRTC realtime — «живой разговор» 🎯 (максимальный UX)

**Как:** постоянный WebRTC-канал между телефоном и OCPlatform.
Вика «на линии» всегда, как разговор по телефону.

**Технологии:**
- LiveKit / Daily.co для WebRTC
- Streaming STT (Whisper streaming / Deepgram)
- Streaming TTS (ElevenLabs streaming / Piper streaming)
- Barge-in — можно перебивать Вику как живого собеседника
- Латентность 200-500мс — почти естественный разговор

**Референс:** уже есть в `voice-agent-platform` на нашем сервере (отдельный репо
Rivega42/voice-agent-platform) — там как раз архитектура под это.

| Плюсы | Минусы |
|-------|--------|
| Максимальный UX «как с человеком» | Сложнее реализовать |
| Barge-in, интонации, паузы | Жрёт батарею |
| Realtime | Требует стабильного интернета |
|  | Трафик 4G |

**Статус:** 🔲 M5+ (долгосрок).

## Handoff между телефоном и домашней Викой

**Ключевая фича**: разговор переходит между surfaces без потери контекста.

**Сценарий:**
1. Едешь в машине, разговариваешь с Викой в наушниках через телефон
2. Приехал домой, зашёл на кухню
3. Домашняя Вика: «продолжим?» → перехват на кухонное устройство
4. Session ID тот же → память не рвётся

**Как это работает технически:**
- Все устройства (телефон + RPi4-satellites) ходят в одну OCPlatform-сессию
- Устройство-победитель arbitration шлёт «я тебя слышу» через MQTT
- Предыдущее устройство отходит в passive
- См. подробнее [MULTI-DEVICE.md — handoff](MULTI-DEVICE.md#5-handoff--переход-разговора-между-комнатами)

**Триггеры перехода:**
- Wake-word на новом устройстве («Вика, я дома»)
- Автообнаружение: телефон подключился к домашнему Wi-Fi → предложить handoff
- Явная команда: «Вика, переключи на кухню»

## Что нужно от OCPlatform

Чтобы всё это работало гладко, OCPlatform должен уметь:

- [ ] Мульти-surface session — одна сессия, несколько источников входа/выхода
- [ ] Fast context switching — быстрое переключение source device (< 500мс)
- [ ] Streaming API — для WebRTC-режима
- [ ] Push notifications на телефон — Вика может инициировать разговор
      («Ром, у тебя встреча через 15 минут, готов?»)

Заложить в архитектурные требования к OCPlatform gateway.
