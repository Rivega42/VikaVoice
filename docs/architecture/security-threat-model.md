# Модель угроз (безопасность)

> Статус: 🟡 design (первая итерация; ревью безопасности — до пилотов, EPIC-8) ·
> Обновлено: 2026-07-07 · Связанные документы: [152-ФЗ](../compliance/152fz.md),
> [privacy-mode](privacy-mode.md), [data-model](data-model.md)

## Активы

1. **Аудиозаписи и стенограммы встреч** — самое чувствительное (NDA, врачебная тайна).
2. **Голосовые отпечатки** — биометрия, необратимая при утечке.
3. **Токены** — OCPlatform, устройств (ingest), OAuth УДЯ.
4. Конфигурация устройства (Wi-Fi-пароль), целостность прошивки/образа.
5. Репутация ключевого обещания «данные не утекают».

## Нарушители

- **Сосед по LAN** (гость офиса/дома, скомпрометированный ноутбук в той же сети).
- Внешний атакующий через интернет (для Cloud-редакции и gateway).
- Физический доступ к устройству (кража microSD/NVMe из переговорки).
- Недобросовестный инсайдер у заказчика (доступ к кабинету).
- Цепочка поставки (подмена образа/зависимостей).

## Архитектурные принципы

1. **Deny by default наружу**: устройство инициирует соединения только к явно
   сконфигурированным адресам (OCPlatform, обновления); входящие — только LAN-сервисы
   (ingest, status, metrics), закрытые токеном там, где данные чувствительны.
2. **Данные и биометрия — раздельные хранилища** с отдельными ключами
   ([data-model](data-model.md)).
3. **Приватный режим = сетевая гарантия, не UI-флаг** ([privacy-mode](privacy-mode.md)).
4. **Никаких секретов и адресов инфраструктуры в репозитории** (правило CONTRIBUTING).
5. Публичная поверхность Cloud-редакции проходит отдельное ревью до запуска SaaS.

## Что сознательно принято сейчас (прототип)

- Скелет ingest работает без auth/TLS **только** в доверенной LAN разработчика; это
  зафиксировано здесь и в [ADR-0009](../adr/0009-ingest-protocol.md) как временное
  состояние. Перевод ADR-0009 в accepted невозможен без закрытия T1.

## Открытые вопросы

- Secure boot / проверка целостности на RPi (ограниченно возможна) — стоит ли для Edge.
- Модель ключей шифрования: TPM-модуль (есть M.2) vs пароль администратора vs ключ-файл.
- Пентест перед первой продажей (внешний подрядчик) — бюджетировать в EPIC-8.
