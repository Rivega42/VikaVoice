# Мобильный клиент — Вика в наушниках через телефон

> Статус: 🟡 design (реализация — EPIC-9) · Обновлено: 2026-07-07 · Связанные ADR:
> [0009 протокол ingest](../adr/0009-ingest-protocol.md)

> Сценарий: телефон + любые наушники (Bluetooth или проводные) = полноценная Вика в кармане.
> Идёшь по улице, едешь в машине, в спортзале — Вика на связи, память та же, что дома.

## Ключевая идея

**Телефон = ещё один VikaVoice-satellite**, наравне с домашними устройствами
(см. [multi-device.md](multi-device.md)). Всё через ту же **OCPlatform-сессию** — Вика
помнит, что ты сказал утром дома, и отвечает в наушниках в машине. Никакого разрыва
контекста между «домом» и «улицей».

## Варианты подключения — от простого к продвинутому

### 1. Telegram голосовыми 🟢 (доступно уже сейчас)

Тапаешь «микрофон» → говоришь → Вика транскрибирует и отвечает (с TTS-директивами — голосом).

| Плюсы | Минусы |
|-------|--------|
| Работает без разработки | Не hands-free (нужен tap) |
| Все наушники поддерживаются | Не realtime, задержка |

**Статус:** ✅ работает через OCPlatform. Использовать как fallback.

### 2. PWA (Progressive Web App) 🌐 — первый шаг EPIC-9

Отдельный сайт-приложение: открыл в браузере телефона → работает; можно «установить»
на главный экран. Технологии: Web Speech API (STT/TTS в браузере), fetch к OCPlatform
gateway, Service Worker.

| Плюсы | Минусы |
|-------|--------|
| Быстро сделать (~1–2 недели) | Wake-word в фоне не работает |
| Никаких сторов, легко обновлять | Только tap-to-talk |
| iOS и Android | Качество Web Speech хуже нативного; ограничения Safari |

**Статус:** 🔲 EPIC-9, этап A.

### 3. Нативное приложение 📱

Flutter или React Native → App Store + Google Play. Wake-word «Вика» в фоне
(лицензию движка см. [ADR-0006](../adr/0006-wake-word.md) — Porcupine коммерческий),
STT/TTS нативные (Speech Framework iOS / Android Speech), мозг — OCPlatform.
Приватный режим: on-device LLM (Gemma 3n / Phi-3.5-mini через MediaPipe/ONNX Runtime)
или деградация «только STT, ответ текстом, в облако не идём».

| Плюсы | Минусы |
|-------|--------|
| Полноценный hands-free, wake-word в фоне | ~2–3 месяца разработки |
| Работает везде, где есть 4G | Публикация и комиссии сторов, поддержка двух ОС |

**Статус:** 🔲 EPIC-9, этап B.

### 4. Bluetooth-гарнитура напрямую к устройству 🎧

Наушники подключаются не к телефону, а к домашнему устройству (дома) или мини-хабу
в кармане (LTE-модем + RPi Zero 2W + BT). Реальный «агент в наушниках» без телефона
в руках, но BT-радиус ~10 м и второе носимое устройство. **Статус:** 🔲 backlog.

### 5. WebRTC realtime 🎯 — максимальный UX

Постоянный канал телефон ↔ OCPlatform: streaming STT/TTS, barge-in (перебивать Вику,
как человека), латентность 200–500 мс. Технологии: LiveKit/Daily, streaming Whisper.
Референс — репозиторий voice-agent-platform (отдельный проект автора с архитектурой
под это). **Статус:** 🔲 EPIC-9, этап C.

## Поэтапный план (внутри EPIC-9)

| Этап | Что | Оценка |
|------|-----|--------|

## Handoff между телефоном и домашней Викой

Разговор переходит между поверхностями без потери контекста: все устройства ходят в одну
OCPlatform-сессию; победитель arbitration публикует «я тебя слышу» в MQTT; предыдущее
устройство уходит в passive. Триггеры: wake-word на новом устройстве, автообнаружение
домашнего Wi-Fi, явная команда «Вика, переключи на кухню». Механика —
[multi-device.md](multi-device.md#5-handoff--переход-разговора-между-комнатами).

## Что нужно от OCPlatform

- [ ] Мульти-surface session — одна сессия, несколько источников входа/выхода
- [ ] Fast context switching — переключение source device < 500 мс
- [ ] Streaming API — для WebRTC-режима
- [ ] Push-уведомления на телефон — Вика может инициировать разговор
      («у тебя встреча через 15 минут, готов?»)

Заложить в архитектурные требования к OCPlatform gateway (контракт — EPIC-5, E5.4).
