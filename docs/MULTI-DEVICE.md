# Multi-Device — несколько Вик как единая система

> Сценарий: 3–5 устройств VikaVoice расставлены по дому (кухня, кабинет, спальня, детская, гостиная). Они должны работать как **одна Вика**, а не как несколько разных.

---

## Ключевые задачи

### 1. Arbitration — кто отвечает?

**Проблема:** Ты говоришь «Вика» в коридоре — тебя слышат 2-3 устройства. Все они не должны отвечать хором.

**Решение (Echo Spatial Perception, ESP):**
1. Все устройства детектят wake-word параллельно
2. Каждое измеряет свой **confidence score**:
   - Амплитуда сигнала (dB)
   - Signal-to-Noise Ratio (SNR)
   - Confidence wake-word модели (openWakeWord/Porcupine возвращает 0–1)
3. В течение ~200мс обмениваются метриками через локальный MQTT
4. Побеждает устройство с максимальным score → **только оно отвечает**
5. Остальные молчат, но продолжают слушать (могут перехватить, если пользователь пошёл в другую комнату)

**Реализация:**
- MQTT-топик `vika/arbitration/wake-word`
- Timeout арбитража: 250мс
- Если сеть упала — каждое устройство отвечает само (fail-safe, лучше чем молчание)

### 2. Единый мозг

**Проблема:** Вика на кухне должна помнить, что я сказал ей в кабинете 5 минут назад.

**Решение:** Все устройства ходят в **одну и ту же OCPlatform-сессию**.

- Session ID общий для всех устройств пользователя
- Контекст, память, инструменты — на стороне OCPlatform
- Локально каждое устройство — только "рот и уши"

**Что это даёт:**
- Продолжение разговора между комнатами: начал на кухне, дошёл до кабинета, договорил
- Общая память задач/напоминаний
- Согласованность личности (одна Вика, а не 3)

### 3. Синхронизация состояний

**Общий state, шарящийся через MQTT retained messages:**

| Состояние | Scope | Пример |
|-----------|-------|--------|
| Приватный режим | **глобальный** | Включён на кухне → красный LED на всех |
| Активный разговор | **локальный** | Только устройство-победитель arbitration |
| Таймер/напоминание | по выбору | «через час» → звенит везде; «на кухне через час» → только там |
| Встреча/фасилитация | **локальный** | Одна встреча — одно устройство (то, что на столе) |
| Домашний режим | **глобальный** | «Вика, я ушёл» → все переходят в passive mode |

**MQTT-топики:**
```
vika/state/private-mode        (retained)
vika/state/home-mode           (retained)
vika/state/active-conversation (не retained, TTL 30s)
vika/timers/#                  (retained per timer)
vika/events/wake-word          (не retained)
```

### 4. Discovery — как устройства находят друг друга

**mDNS/Zeroconf** — плюс автоматика:
- Каждое устройство анонсит: `_vika._tcp.local` с полями `role=hub|satellite`, `room=kitchen`
- Satellites сканят и находят hub автоматически
- Hostname: `vika-kitchen.local`, `vika-office.local` и т.д.

**Первое устройство в сети автоматически становится hub'ом.**
Если приходит новое устройство и hub уже есть — становится satellite.
Если hub пропал — leader election по MAC-адресу или uptime.

### 5. Handoff — переход разговора между комнатами

**Сценарий:** Начал говорить с Викой на кухне, пошёл в кабинет, продолжаешь.

**Механика:**
1. Устройство на кухне детектит "пользователь ушёл" (VAD показывает тишину > 3 сек)
2. Устройство в кабинете детектит "новый голос" + wake-word (или продолжение фразы)
3. Через arbitration кабинет побеждает
4. **Session ID** тот же → OCPlatform продолжает контекст без разрыва
5. Ответ звучит уже из кабинета

**Что нужно:**
- Voice fingerprinting — узнавать голос Романа vs дети/гости
- Session pinning в OCPlatform — быстрое переключение source device
- Плавный переход: последняя реплика "перевозится" (текст, не аудио)

### 6. Multi-room audio (позже)

Отдельная задача — синхронное воспроизведение музыки/подкастов на нескольких устройствах.

**Кандидаты:**
- **Snapcast** — Linux-native, синхронизация ~1мс, работает через ту же локальную сеть
- **PulseAudio RTP** — проще, менее точная синхронизация
- **AirPlay 2** — если хотим совместимости с Apple

Для голоса Вики (её ответов) — обычно играет только device-победитель. Но можно опционально включать «broadcast» режим («Вика, скажи всем: обедать!»).

## Архитектура

```
                    ┌─────────────────────────┐
                    │   OCPlatform Gateway    │
                    │   (сервер, интернет)    │
                    │                         │
                    │  Единая сессия Вики,    │
                    │  память, инструменты    │
                    └────────────┬────────────┘
                                 │ HTTPS
                                 │
        ┌────────────────────────┼────────────────────────┐
        │                        │                        │
    ┌───▼─────┐             ┌────▼────┐              ┌────▼────┐
    │ vika-   │◄──MQTT─────►│  vika-  │◄──MQTT──────►│  vika-  │
    │ kitchen │             │ office  │              │ bedroom │
    │ [HUB]   │             │[satellite]             │[satellite]
    └─────────┘             └─────────┘              └─────────┘
                    локальная сеть 192.168.x.x
                    mDNS discovery, MQTT coordination
```

**Роли устройств:**

- **Hub** — первое устройство или самое мощное:
  - Запускает MQTT broker (mosquitto)
  - Координирует arbitration
  - Хранит retained state
  - Роль эфемерная — при падении, следующее устройство становится hub

- **Satellite** — стандартное устройство:
  - Полный VikaVoice-стек (wake-word, STT, TTS)
  - Подключено к MQTT hub'а
  - Может стать hub'ом, если hub упал

**Все устройства имеют идентичную прошивку.** Роль определяется в рантайме.

## Сценарии

### Сценарий 1: "Вика" в коридоре, слышат 2 устройства
1. Кухня: confidence 0.87, кабинет: 0.64
2. Кухня выигрывает arbitration → отвечает
3. Кабинет остаётся в режиме listening (может перехватить)

### Сценарий 2: Приватный режим — приехал клиент к психологу
1. «Вика, приватный режим на 2 часа»
2. Локальное устройство обрабатывает команду
3. Публикует `vika/state/private-mode = true` (retained)
4. **Все** устройства подхватывают → красный LED, локальные LLM активированы, сеть отрублена
5. Через 2 часа таймер снимает флаг → все выходят из режима

### Сценарий 3: Переход из комнаты в комнату
1. Кухня: «Вика, поставь таймер на 15 минут для яиц»
2. Пользователь идёт в кабинет
3. Через 15 минут: звенит **та же кухня** (таймер локальный "kitchen")
4. Если бы сказал «поставь таймер, я через 15 минут выхожу» — звенело бы на всех (глобальный)

### Сценарий 4: Broadcast команда
- «Вика, скажи всем: ужин готов»
- Hub рассылает `vika/broadcast/announce` во все устройства
- Все проговаривают синхронно через Piper

## Что нужно проверить/решить позже

- **Voice fingerprinting** — умеет ли Whisper отличать Романа от Леры? Или нужна отдельная модель?
- **Latency arbitration** — 250мс достаточно, или пользователь заметит задержку?
- **Приватный режим — распространение** — все устройства должны иметь **локальную LLM** (то есть на всех должна быть достаточная RAM/CPU). Иначе satellite без LLM в приватном режиме = молчащая коробка.
- **Обновления прошивки** — как выкатывать OTA на N устройств одновременно?
- **Начальный setup** — второе устройство должно "увидеть" hub и подключиться автоматически через тот же AP-режим или уже через локальный MQTT.
