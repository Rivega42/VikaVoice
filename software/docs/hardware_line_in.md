# Аппаратный захват системного звука (проводом)

Задача: подать звук онлайн-встречи с ПК/ноутбука в наше устройство физически,
без установки ПО на ПК. Устройство подмешивает этот звук к сигналу микрофонного
массива и транскрибирует всё вместе.

Зачем нужен, когда есть программный клиент: там, где на ПК нельзя ставить софт
(чужой/корпоративный ноутбук, режимные объекты), а также как самый простой и
надёжный путь без прав администратора.

## Варианты подключения (по возрастанию качества)

1. **3.5 мм: выход наушников ПК → линейный вход устройства.**
   Простейший путь. Минус: на ПК звук уходит в устройство, локально может пропасть —
   решается разветвителем (сплиттер), чтобы звук шёл и в колонки/наушники, и в устройство.

2. **USB-аудио адаптер с линейным входом на устройстве.**
   К нашему устройству (Raspberry Pi) подключается USB-звуковая карта с line-in;
   в неё — кабель с выхода ПК. Даёт отдельный аудиоканал, не мешая микрофонному массиву.
   Рекомендуемый путь для прототипа.

3. **HDMI-аудио экстрактор.**
   Если ПК выводит на внешний экран по HDMI — экстрактор ответвляет цифровой звук
   (S/PDIF или аналог) в устройство. Чистый сигнал, но громоздко.

4. **USB-«петля» (устройство как USB-звуковая карта для ПК).**
   Устройство представляется для ПК как USB-гарнитура; ПК сам направляет в него звук.
   Требует USB-gadget-режима (на Pi реализуемо) — путь для серийной версии.

## Обработка на устройстве

- Линейный вход = отдельный аудиоканал наравне с каналами массива.
- Микс: канал «системный звук» помечается как отдельный источник и может идти
  в стенограмму как «Удалённый участник (ПК)», а не смешиваться с локальными голосами —
  это лучше для диаризации.
- Уровни: предусмотреть аппаратный/программный регулятор усиления линейного входа,
  т.к. выходы ПК сильно различаются по громкости.

## Мини-BOM аппаратного пути (ориентир)

| Позиция | Назначение | ~$ |
|---|---|---|
| USB-звуковая карта с line-in | Линейный вход в устройство | 8–15 |
| Аудиокабель 3.5 мм + сплиттер | ПК → устройство, с сохранением локального звука | 4 |
| (опц.) HDMI-аудио экстрактор | Цифровой звук с HDMI | 15–25 |

## Ключевой нюанс

Юридически звук встречи с ПК — те же переговоры: уведомление о записи и согласие
участников обязательны (152-ФЗ), независимо от способа захвата.
